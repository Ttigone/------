基于 Linux 和 Web 的视频监控系统的设计与实现
一、项目概述
本设计以嵌入式 Linux 系统（RK3576 平台）为核心，聚焦 “实用化和易落地” 目标，开发了一套集 USB 摄像头实时监控、双重人体检测、智能安防联动、远程数据查看于一体的系统。系统通过 Linux 标准 V4L2 驱动完成 USB 摄像头视频采集; 内置 Boa 轻量级 Web 服务器，用户可通过局域网直接查看实时画面，或通过 Qt APP 查看传感器数据、调节安防阈值。
二、硬件系统设计
核心元器件围绕 “视频采集 + 双重感知 + 控制响应” 三大需求配置，具体如下：
模块类型	具体元器件	核心作用
主控模块	RK3576 嵌入式开发板	运行 Linux 系统，承载 V4L2 摄像头驱动、Boa 服务器、安防逻辑处理及 YOLO 模型推理
感知模块	USB 摄像头	采集实时视频数据，为 YOLO 模型提供图像输入，通过 V4L2 驱动与 RK3576 通信
	BH1750 光照传感器	实时采集环境光照值，数据同步至 APP 供查看
	PIR 人体红外传感器	快速检测人体活动，作为 YOLO 识别模型启动触发信号，低功耗运行
	磁簧开关传感器	检测门窗开合状态，触发双重人体检测流程激活
控制模块	声光报警模块（蜂鸣器 + LED 灯）	安防条件满足时，通过声音与灯光提示异常
存储模块	RK3576 板载存储 SD 卡	存储安防触发时的视频录像和系统日志
通信模块	RK3576 板载 WiFi 模块 	实现局域网通信，支撑Boa Web服务与视频流传输

三、核心功能与实现逻辑
系统设计自动安防、手动控制、远程查看三种核心工作模式，通过本地按键或手机 APP 切换，各模式逻辑独立且状态双向同步。
1. 自动安防模式（系统默认启动）
触发方式: 系统上电后自动进入，无需手动干预，适配日常无人值守场景。
工作逻辑:
1.	传感器数据采集与触发：BH1750 传感器通过 I2C 接口循环采集光照值；磁簧开关通过 GPIO 实时监测门窗状态，默认处于 “低电平监测” 状态；当磁簧开关检测到 “门窗开启”时，RK3576 立即激活 PIR 人体红外传感器。
2.	双重人体检测流程：第一重（快速触发）：PIR 传感器检测到人体活动，向系统发送 “疑似有人” 信号，触发 USB 摄像头启动与 YOLO 模型加载，避免 YOLO 持续运行占用资源；第二重（精准验证）：YOLO 轻量化模型对 USB 摄像头采集的实时视频帧进行推理，识别画面中是否存在人体（置信度阈值设为 0.6，高于阈值判定为 “确认有人”）；仅当 PIR 触发且 YOLO 验证通过时，判定为 “安防异常”，避免单一传感器误报（如宠物移动、物体晃动触发 PIR）。
3.	安防联动响应：触发声光报警模块启动（蜂鸣器响、LED 每秒闪烁 2 次）；调用视频录制工具捕获 USB 摄像头视频（1080P，时长 10 秒，文件按时间戳命名，存储路径固定为/home/record/）报警触发后，手动按下 RK3576 开发板物理按键可停止声光报警，视频文件保留在本地供后续查看。
4.	状态同步：门窗状态、人体检测状态（“无人”“疑似有人”“确认有人”）、光照值、报警状态实时上传至手机 APP，异常状态以红色字体醒目提示。
2. 手动控制模式
触发方式: 通过手机 APP “模式切换” 按钮选择 “手动模式”，适配特殊监控需求（如临时调试、应急操作）。
工作逻辑: 
1.	手动控制报警：APP 端提供 “触发报警”“停止报警” 按钮，操作指令通过 TCP 协议实时传输至 RK3576；
2.	手动录制视频：APP 端点击 “开始录制”，可手动触发 USB 摄像头视频录制（时长 10-60 秒可选），录制状态同步显示；
3.	状态反馈：所有手动操作结果实时回显至 APP，避免操作延迟或失效。
3. 远程查看与阈值调节模式
触发方式: 设备连接局域网后，通过手机 APP 切换至 “远程模式”，适配用户远程监控场景。
工作逻辑: 
1.	实时数据查看：APP 通过 TCP 协议接收 RK3576 上传的传感器数据，以文本形式展示环境参数（光照值）与安防状态（门窗状态、人体检测状态、报警状态）；
2.	阈值调节：支持远程修改 “安防触发时长”（默认 10 秒，可在 5-30 秒区间调整）与 YOLO 置信度阈值（默认 0.6，可在 0.4-0.8 区间调整）,调节后参数实时同步至 RK3576；
3.	视频查看跳转：APP 点击 “查看监控” 按钮，自动调用手机浏览器打开 Web 页面（地址为 RK3576 的局域网 IP）,加载 USB 摄像头实时画面。
四、软件设计框架
软件基于 Linux 系统采用模块化编程思想，核心流程拆解为初始化、数据采集、双重人体检测、模式控制、报警联动、交互通信六大模块。
1. 初始化模块（系统启动即执行）
功能目标: 完成硬件外设、软件服务与模型文件的初始化，为后续功能运行奠定基础。
实现逻辑: 
1.	硬件外设初始化：通过 Linux 系统调用初始化 GPIO（控制 PIR 传感器、磁簧开关、声光报警模块）、I2C（驱动 BH1750 传感器）、USB 接口（识别 USB 摄像头）；配置 V4L2 驱动，设置摄像头采集格式；
2.	软件服务初始化：启动 Boa Web 服务器，加载配置文件并指定网页目录与监听端口；启动MJPG-streamer服务，绑定摄像头采集数据并生成视频流；
3.	模型初始化：加载 YOLO 轻量化模型的 ONNX 格式文件，初始化推理环境，设置置信度阈值，等待触发信号。
2. 数据采集模块
功能目标: 稳定获取 USB 摄像头视频数据与各传感器状态数据，为检测与交互提供数据支撑。
实现逻辑: 
1.	视频数据采集：通过 V4L2 标准接口循环读取摄像头帧数据，按"缓冲区入队→帧数据出队→传输至MJPG-streamer/检测模块"流程处理，每帧数据处理延迟控制在50ms以内；
2.	传感器数据采集：光照数据通过 I2C 工具包读取 BH1750 寄存器值并转换为 lux 单位；门窗与 PIR 状态通过 Linux GPIO 子系统读取对应引脚电平，电平变化时立即触发状态更新。
3. 双重人体检测模块
功能目标: 结合 PIR 传感器与 YOLO 模型，实现 “快速触发 + 精准识别”，降低误报率，兼顾效率与精度。
实现逻辑: 
1.	触发判断：仅当磁簧开关检测到 “门窗开启” 且 PIR 传感器检测到 “疑似有人” 时，启动 YOLO 模型推理，避免无效消耗；
2.	精准识别：从MJPG-streamer获取视频帧，输入YOLO模型进行快速推理（单帧推理时间≤100ms），输出检测结果（是否含人体、置信度）； 
3.	结果融合：若 YOLO 检测置信度≥预设阈值（默认 0.6），判定为 “确认有人”，向报警联动模块发送异常信号；若置信度＜阈值，判定为 “误触发”，返回监测状态，关闭 YOLO 模型以节省资源。
4. 模式控制模块
功能目标: 根据用户指令切换工作模式，执行对应控制逻辑，确保模式切换无冲突、状态一致。
实现逻辑: 
1.	模式判断：通过 Linux 信号量机制监听 APP 指令或本地按键电平，确定当前工作模式；
2.	逻辑执行：自动模式下启动传感器联动与双重人体检测逻辑，禁用手动控制指令；手动模式下优先响应 APP 手动指令，暂停传感器联动（保留数据采集）；远程模式下同步启用数据上传与 Web 视频服务，确保远程访问流畅。
5. 报警与联动模块
功能目标: 接收异常信号后，快速触发报警与视频录制，确保异常事件可追溯。
实现逻辑: 
1.	异常响应：接收双重人体检测模块的 “确认有人” 信号后，立即启动声光报警模块；
2.	视频录制：指定录制时长（10 秒）与存储路径，同步捕获报警时段视频；
3.	复位机制：报警触发后，检测到 “门窗关闭” 或 “手动按键按下” 时，停止报警与录制，恢复默认监测状态。
6. 交互与通信模块
功能目标: 实现 Web 端视频访问与 APP 端数据交互，确保远程操作实时响应。
实现逻辑
1.	Web服务：Boa Web 服务器监听 80 端口，浏览器请求 RK3576 的 IP 地址时，返回包含MJPEG视频流、状态显示的静态网页；
2.	APP 端通信：基于 TCP 协议建立 APP 与 RK3576 的长连接，实现模式切换指令、阈值调节参数的下行传输，以及传感器数据、人体检测状态、设备状态的上行反馈；
3.	视频流传输： MJPG-streamer服务独立运行（默认8080端口），Web页面通过<img>标签或JavaScript定时刷新方式加载MJPEG流，实现视频实时播放。
五、Web 与 APP 端设计
1. Web 端（基于 Boa 服务器）
核心定位:  提供一体化的视频监控解决方案，用户仅需浏览器即可完成查看监控操作。
实现方案: 视频监控区域，居中显示实时视频流，基于MJPEG方案，支持全屏显示，适配不同分辨率的显示设备。
2. APP 端（基于 Qt 开发）
核心定位: 实现传感器数据查看、模式切换、阈值调节与 Web 视频跳转，完整呈现系统核心功能。
实现方案: 界面采用简洁布局，分为数据展示区、模式控制区、阈值调节区、视频跳转区四大模块：
1.	数据展示区：以文本框分类显示光照值、门窗状态、人体检测状态（“无人”“疑似有人”“确认有人”）、报警状态，异常状态标红；
2.	模式控制区：提供 “自动”“手动”“远程” 三个单选按钮，手动模式下显示 “触发报警”“停止报警”“开始录制” 按钮；
3.	阈值调节区：提供两个滑块组件，分别调节 “安防触发延迟”（5-30 秒）与 “YOLO 置信度阈值”（0.4-0.8），滑动后自动校验范围，有效值实时同步至设备；
4.	视频跳转区：居中放置 “查看监控” 按钮，点击后调用手机默认浏览器打开 Web 视频页面，实现一键跳转。

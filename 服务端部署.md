# moe_server 服务端完整部署方案

---

## 一、关于 kickpi demo 能否直接使用

| Demo | 能否用于 USB V4L2 实时流 | 说明 |
|---|---|---|
| `rknn_yolov5_demo` | ❌ 不能直接用 | 处理**静态图片文件**，不接入摄像头 |
| `rknn_yolov5_video_demo` | ❌ 不能直接用 | 使用 **MPP 硬件解码器**处理 H264/H265 **文件**，无法读取 V4L2 实时流 |
| **moe_server（本方案）** | ✅ 可用 | V4L2 直接抓 MJPEG 帧 → RKNN 推理 → 推流 |

**正确做法**：从 SDK demo 源码中**复制** `postprocess.h / postprocess.cc` 的后处理逻辑（已在 `yolo_detect.cpp` 中内联简化实现），使用相同的 `.rknn` 模型文件，调用 RKNN C API 做推理。

## 二、整体架构

```
USB摄像头 /dev/video0
        │  V4L2 MJPEG
        ▼
 ┌─────────────────────────────────────────────────────┐
 │                   moe_server 守护进程                │
 │                                                     │
 │  capture_thread                                     │
 │    ├─ V4L2 抓帧 (MJPEG 640×480 @25fps)             │
 │    ├─ [YOLO ON] → RKNN 推理 → 画框 → JPEG          │
 │    ├─ mjpeg_push_frame() → g_frame 帧缓冲           │
 │    └─ [录像中] → 写 JPEG 帧到 /home/record/         │
 │                                                     │
 │  mjpeg_server_thread  :8091                         │
 │    └─ 每个浏览器/APP连接一线程，推 multipart MJPEG  │
 │                                                     │
 │  tcp_server_thread    :12345                        │
 │    └─ 多手机APP同时连接，JSON命令/事件双向通信      │
 │                                                     │
 │  sensor_thread                                      │
 │    └─ BH1750(I2C) + PIR + 磁簧开关(GPIO) → 广播    │
 └─────────────────────────────────────────────────────┘
        │                         │
        ▼ :8091                   ▼ :12345
  Boa index.html            手机 APP (多个)
  <img src=":8091/...">     收发 JSON 事件
```

## 三、板子端依赖安装

```bash
# 在 RK3576 板子上（或交叉编译 sysroot 中）安装依赖
sudo apt update
sudo apt install -y \
    libopencv-dev \      # OpenCV (含 imgproc/imgcodecs)
    cmake g++ make

# 确认 i2c-tools（调试用）
sudo apt install -y i2c-tools

# 确认摄像头节点
ls /dev/video*
v4l2-ctl --list-devices

# 确认摄像头支持 MJPEG 格式
v4l2-ctl -d /dev/video0 --list-formats-ext | grep -i mjpeg
```

## 四、编译步骤

### 方法 A：直接在板子上编译（最简单）

```bash
# 1. 将 moe_server/ 源码传到板子
scp -r ./moe_server kickpi@192.168.x.x:/home/kickpi/

# 2. 在板子上编译
ssh kickpi@192.168.x.x
cd /home/kickpi/moe_server

mkdir build && cd build
cmake .. \
  -DRKNN_SDK_DIR=/home/kickpi/rknn_yolov5_demo_Linux \
  -DRKNN_INCL_DIR=/home/kickpi/rknn_yolov5_demo_Linux/include
  # 注意：rknn_api.h 在 SDK 的 include/ 或 rknpu2/include/ 下

make -j4
```

> **rknn_api.h 路径确认**：
> ```bash
> find /home/kickpi/rknn_yolov5_demo_Linux -name "rknn_api.h"
> # 典型路径: rknn_yolov5_demo_Linux/include/rknn_api.h
> ```

### 方法 B：PC 端交叉编译（速度更快）

```bash
# PC 端（Ubuntu）安装工具链
# 工具链路径参考 kickpi 文档: SDK/prebuilts/gcc/.../aarch64-none-linux-gnu

# 还需要交叉编译版 OpenCV（或用板子上的 sysroot）
# 简化方案：先在板子上编译 OpenCV，再 scp 回来

mkdir build && cd build
cmake .. \
  -DCMAKE_TOOLCHAIN_FILE=../toolchain-aarch64.cmake \
  -DRKNN_SDK_DIR=/path/to/rknn_yolov5_demo_Linux \
  -DRKNN_INCL_DIR=/path/to/rknpu2/include \
  -DOpenCV_DIR=/path/to/opencv-aarch64/lib/cmake/opencv4

make -j$(nproc)

# 传到板子
scp build/moe_server build/librknnrt.so kickpi@192.168.x.x:/home/kickpi/moe_server/
```

## 五、运行

```bash
# 在板子上
cd /home/kickpi/moe_server

# 设置 RKNN 库路径
export LD_LIBRARY_PATH=.:$LD_LIBRARY_PATH

# 启动（参数1: 摄像头设备  参数2: RKNN 模型路径）
./moe_server /dev/video0 \
  /home/kickpi/rknn_yolov5_demo_Linux/model/RK3576/yolov5s-640-640.rknn

# 预期输出:
# [v4l2] format: MJPEG 640x480
# [v4l2] streaming /dev/video0 @25fps
# [yolo] output[0] scale=0.003922 zp=-128
# [yolo] model loaded: .../yolov5s-640-640.rknn
# [mjpeg] listening on :8091
# [tcp] listening on :12345  (max 16 clients)
# [sensor] started
# [alarm] GPIO buzzer=149 led=150 initialized
# [main] all threads started
```

## 七、Boa 网页配置（视频嵌入）

修改 `/var/www/index.html`，将视频标签改为指向 `moe_server` 的 MJPEG 端口：

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>智能监控</title>
  <style>
    body { background:#111; color:#eee; font-family:sans-serif; text-align:center; }
    img  { max-width:100%; border:2px solid #4caf50; border-radius:8px; margin-top:20px; }
  </style>
</head>
<body>
  <h2>实时监控画面</h2>
  <!-- 替换 192.168.x.x 为板子实际 IP -->
  <img src="http://192.168.x.x:8091/?action=stream"
       alt="loading stream...">
</body>
</html>
```

访问方式：
- **网页监控**：浏览器打开 `http://板子IP:80`（Boa 服务）
- **视频直连**：浏览器/VLC 打开 `http://板子IP:8091/?action=stream`
- **APP 控制**：手机 App 连接 `板子IP:12345`（TCP JSON）

## 八、开机自启（systemd）

```bash
sudo nano /etc/systemd/system/moe_server.service
```

```ini
[Unit]
Description=Moe Security Server
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/home/kickpi/moe_server
ExecStart=/home/kickpi/moe_server/moe_server /dev/video0 /home/kickpi/rknn_yolov5_demo_Linux/model/RK3576/yolov5s-640-640.rknn
Environment=LD_LIBRARY_PATH=/home/kickpi/moe_server
Restart=on-failure
RestartSec=3

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable moe_server
sudo systemctl start  moe_server
sudo journalctl -u moe_server -f   # 查看日志
```

## 九、录像文件转换（可选）

`moe_server` 录像格式为连续 JPEG 帧（.mjpeg），可用 ffmpeg 转为 mp4：

```bash
# 单文件转换
ffmpeg -f mjpeg -i /home/record/alarm_1709876543.mjpeg \
       -c:v libx264 -pix_fmt yuv420p \
       /home/record/alarm_1709876543.mp4

# 批量转换脚本
for f in /home/record/*.mjpeg; do
    ffmpeg -f mjpeg -i "$f" -c:v libx264 "${f%.mjpeg}.mp4" && rm "$f"
done
```

## 十、文件结构总览

```
moe_server/
├── CMakeLists.txt          # 构建配置
├── toolchain-aarch64.cmake # 交叉编译工具链（可选）
├── state.h                 # 全局帧缓冲 + 原子状态
├── main.cpp                # 主入口，启动各线程
├── v4l2_capture.hpp/.cpp   # V4L2 USB 摄像头 MJPEG 采集
├── yolo_detect.hpp/.cpp    # RKNN 推理 + YOLOv5 后处理 + OpenCV 画框
├── mjpeg_server.hpp/.cpp   # MJPEG HTTP 流服务 :8091
├── tcp_server.hpp/.cpp     # 手机 APP TCP 命令/事件服务 :12345
├── sensor.hpp/.cpp         # BH1750 光照 + PIR + 门窗传感器
└── alarm.hpp/.cpp          # 蜂鸣器/LED/录像控制
```
